# PriceModule

Данный модуль отвечает за периодическое получение и обновление в базе данных информации о ценах товаров. Также содержит функционал для проверки соединения с БД.

Данный модуль включает в себя следующие контроллеры: PriceController, HealthCheckController.
Каждый из контроллеров содержит в себе методы для возврата ответов при успешном выполнении или возникновении ошибок (success и error).
Данные методы наследуются от абстрактного класса AbstractController.

## PriceController

Данный контроллер предоставляет функционал получения цен.

*****

### Endpoints

    | Метод | URL | Описание |
    | --- | --- | --- |
    | GET | `/api/v1/prices/by-external-id/{externalId}` | Получение информации о ценах |

### Описание функционала модуля

1. **Получение информации о ценах**
`GET /api/v1/prices/by-external-id/{externalId}`

*Входные данные:*
- externalId: string - внешний идентификатор продукта из 1С.

*Функционал:*
Данные о ценах продукта запрашиваются из сервиса 1С автоматически, с интервалом 1 раз в 30 минут. 
Получение данных происходит по ключу product_external_id - внешний id продукта.
Внешний id продукта хранится в базе данных price service, таблица price.
Всего price service связан с двумя таблицами, price и price_promo. Данные таблицы связаны между собой отношением один ко многим.
Если информация получена успешно, то происходит обновление таблиц price и price_promo базы данных price service на основании тех данных, которые были получены.
Для использования на сайте запрашивается информация о ценах из базы данных (таблицы price, price_promo).
Если информация из 1С не была получена, выбрасывает исключение `BusinessException` с сообщением 'Price not found!' и кодом ответа 404 (Not Found).


*Выходные данные:*
Объект с информацией о ценах:
```json
{
    string $productExternalId // внешний id продукта 
    float $promoRetailPrice, // Цена с учетом акций
    float $sellPrice, // Цена продукта
    float $finalSellPrice, // Финальная цена продукта
    int $vatType = 0, // Тип НДС
}
```

## HealthCheckController

Данный контроллер предназначен для проверки соединения с БД.

*****

### Endpoints

HealthCheckController предоставляет следующие эндпоинты:

| Метод | URL | Описание |
| --- | --- | --- |
| GET | `/health` | Получение информации о статусе соединения с БД |

### Описание функционала модуля

**Проверка подключения к БД**
`GET /health`

*Входные данные:*
Объект connection.

*Функционал:*
Происходит проверка подключения к базе данных price-service.
Возникающие при вызове метода connect ошибки перехватываются. В этом случае метод возвращает ответ с информацией об ошибке. 
При успешном подключении возвращается ответ с информацией об успешном подключении.

*Выходные данные:* 
```json
{
    'success' => true,
    'data' => $data,
    'message' => null,
}
```
