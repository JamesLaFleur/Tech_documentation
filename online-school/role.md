# RoleModule

Данный модуль предназначен для управления ролями пользователей. Содержит следующий функционал: получение, создание, обновление и удаление ролей. Доступны следующие роли:
- SUPERADMIN (суперадминистратор) - создание пользователей, назначение ролей, полномочий
- ADMIN (администратор) - функции данного пользователя регулируемые. Их назначает superadmin
- TEACHER (учитель) - admin назначает учителю программы, класс. Назначенные ему уроки учитель может увидеть в расписании, там же будет ссылка на zoom-конференцию. Учителю поступают выполненные учеником задания, которые он должен будет проверить. Также у учителя есть возможность оставлять обратную связь о классе после онлайн-уроков, данный отзыв будет доступен родителям для ознакомления.
- STUDENT (ученик) - может выбрать тариф «Школа» или тариф «Индивидуальный». После того, как за ним будут закреплены учебные программы, ученик сможет ознакомиться с ними. Программы закрепляются либо за учеником, либо за всем классом. Если ученик только что зарегистрировался, то он сможет увидеть все программы, но только их демо-версии. Ученик также может посмотреть состав класса, учителей и свою успеваемость. Ученику будут приходить задания (контрольные, тестирования) от учителя. После проверки, может увидеть оценку и комментарии от учителя. Статусы задания отслеживаются в успеваемости. Ученик видит закрепленное за ним расписание и уроки. Также ученику доступны чаты, их может быть несколько.
- PRESCHOOLER (редактор контента) - создает и редактирует программы и материалы уроков.
- CURATOR (куратор) - куратора закрепляют за классом, он отвечает на вопросы учеников. Может просматривать оценки, без возможности редактирования. Может добавлять, создавать группы для дополнительных занятий.  Есть возможность создания чатов, а также подтверждения или отклонения обратной связи от учителя.
- PARENT (родитель) - включает в себя функционал доступный ученику, кроме возможности выполнения и отправки задания учителю.

### Endpoints

Модуль предоставляет следующие эндпоинты:

| Метод | URL | Описание |
| --- | --- | --- |
| GET | `platform/:platformId/role` | Получение списка ролей |
| GET | `platform/:platformId/role/:roleId` | Получение роли по ID роли |
| POST | `platform/:platformId/role` | Создание новой роли |
| PUT | `platform/:platformId/role/:roleId` | Обновление роли по ID роли |
| DELETE | `platform/:platformId/role/:roleId` | Удаление роли по ID роли |


---

### Объект Role

```json
{
  name: string;
  code: string;
  fields: Record<string, unknown>[] | null;
  fieldsView: Record<string, unknown> | null;
  isPlatformAdmin: boolean;
  permissions: RolePermission[];
  sortIndex: number | null;
  userTariffOptions: UserTariffOption[];
  isDefault: boolean;
}
```

### Описание функционала модуля

1. **Получение списка ролей**
`GET platform/:platformId/role`

*Входные данные:* -

*Функционал:*
Происходит поиск ролей. Возвращает отсортированный по возрастанию значения поля sortindex массив ролей.

*Выходные данные:* массив [Role](#объект-role)[]

2. **Получение роли по ID роли**
`GET platform/:platformId/role/:roleId`

*Входные данные:*
- roleId: string - id роли, динамический параметр, отправляется в url запроса

*Функционал:*
Получение роли по переданному id роли. Если id роли передан некорректно, возвращается null.
Если роль не найдена выбрасывает исключение: NotFoundException
Если данные корректные, возвращает объект Role.

*Выходные данные:* объект [Role](#объект-role)

3. **Создание новой роли**
`POST platform/:platformId/role`

*Входные данные:*
- Тело запроса: объект CreateRoleDto, содержащий поля:
    | Параметр | Тип данных | Валидация | Описание |
    | --- | --- | --- | --- |
    | name | string | Обязательный | Имя |
    | code | string | Обязательный | Код |
    | isPlatformAdmin | boolean | Обязательный | Определяет, является ли данный пользователь администратором |
    | permissions | RolePermissionDto[] | Обязательный | Разрешение |
    | userId | Record<string, unknown>[]| Опциональный | ID пользователя |

*Функционал:*
Происходит создание роли с переданными в CreateRoleDto данными и её сохранение.
Если в процессе создания роли возникают ошибки, то выбрасывается исключение: BadRequestException.
Возвращает созданный объект роли.

*Выходные данные:* объект [Role](#объект-role)

4. **Обновление роли по ID роли**
`PUT platform/:platformId/role/:roleId`

*Входные данные:*
- roleId: string - id роли, динамический параметр, отправляется в url запрос
- Тело запроса: объект UpdateRoleDto, содержащий поля объекта CreateRoleDto

*Функционал:*
Происходит получение роли по переданному id роли. 
На основании переданных значений обновляем объект роли и сохраняем данные.
При возникновении ошибки выбрасывается исключение.

*Выходные данные:* обновлённый объект [Role](#объект-role)


5. **Удаление роли по ID роли**
`DELETE platform/:platformId/role/:roleId`

*Входные данные:*
- roleId: string - id роли, динамический параметр, отправляется в url запрос

*Функционал:*
Получение и удаление роли по переданному id.

*Выходные данные:* -
