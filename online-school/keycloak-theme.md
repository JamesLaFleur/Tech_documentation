# KeyCloak-theme

Модуль для создания кастомных тем KeyCloak. Каждое приложение платформы имеет свой стиль, поэтому базовый интерфейс KeyCloak переопределяется через систему тем.

## Docker

При добавлении новой темы нужно пробросить путь к ней в volume при сборке.

```env
BACKEND_API_URL=https://api.dev.k8s.ponimayu.qtim.ru
```

Адрес бэкенда, к которому обращается фронтенд. Для разработки: установить зависимости, собрать обе темы, поднять docker. Шаблон собирается на yarn + Node 16.20.2.

## Providers

Папка `providers` содержит файл с настройкой OAuth-провайдеров — позволяет авторизоваться через соцсети.

В `docker-compose` volume:

```yaml
- ./providers:/opt/keycloak/providers
```

## Темы (themes)

Папка `themes` хранит собранные темы. Webpack по команде `yarn build` / `yarn build:kz` собирает тему и кладёт её в `themes` в формате, который принимает KeyCloak.

Активные темы:

| Команда | Тема | Платформа |
|---------|------|-----------|
| `yarn build` | ponimau | РФ |
| `yarn build:kz` | empatia | РК |

---

## Webpack

Конфиг хранит переменные с именами тем для KeyCloak. Тема выбирается по параметру `country` — при смене темы меняется переменная, а вместе с ней политика конфиденциальности и логотипы. Тема по умолчанию: `ponimau`.

Webpack собирает отдельную тему для каждой страницы (один `index.ts` на страницу). В static проксируются шрифты, картинки и документы (политики конфиденциальности).

> **Hot reload в dev-режиме не работает.** KeyCloak требует собранного проекта, поэтому после любого изменения нужно пересобирать:
>
> ```bash
> yarn build       # ponimau
> yarn build:kz    # empatia
> ```

## Static

### Messages

Сообщения для каждого языка платформы. Для корректного отображения в файлах нужно указывать кодировку.

### Базовый шаблон

Использует FreeMarker для серверного рендеринга HTML. Работа с шаблонизатором настроена через js-скрипт, ссылающийся на view. Шаблон может расширяться.

Переменные из KeyCloak (`script id="environment"`) доступны во всех страницах:

| Поле | Описание |
|------|----------|
| `permissions` | Действия, доступные пользователю (настраиваются в realm) |
| `labels` | Сообщения для платформы из `static/messages` |
| `validations` | Правила валидации форм |
| `errors` | Тексты ошибок |
| `user` | Данные пользователя |
| `auth` | Данные авторизации |
| `totp` | Настройки двухфакторной аутентификации |
| `env` | Переменные окружения |
| `locales` | Доступные языки |
| `message` | Системные сообщения |
| `social` | Провайдеры соцсетей |

Все поля описаны в `@types/environment`. При добавлении нового текста нужно добавлять его в `types/environment.dts`.

С помощью `<#nested "scripts">` в шаблон можно добавлять дополнительный функционал.

### Theme.properties

Файл `theme.properties` задаёт свойства темы: url, доступные языки, родительская тема (от которой наследуются базовые свойства).

## Views

Папка `views` содержит наборы шаблонов страниц. KeyCloak вызывает нужный шаблон в зависимости от действия пользователя.

Каждый шаблон состоит из двух файлов:

| Файл | Роль |
|------|------|
| `index.ftl` | FreeMarker-шаблон. Импортирует базовый шаблон, подключает скрипт, вставляет URL и данные окружения при серверном рендеринге |
| `index.ts` | Создаёт Vue-приложение, получает переменные окружения из шаблона через `provide`, передаёт их в компоненты |

Большинство шаблонов — базовый функционал KeyCloak, стилизованный под тему платформы. Страницы `login` и `register` содержат расширенный функционал.

### Страница login

- Компонент `<auth-form>` с полями email и пароль
- Валидация через объект `validations`; отправка блокируется при пустых полях
- Метод POST для отправки данных
- Поддержка входа через соцсети
- Ссылки на восстановление пароля и регистрацию (данные из хука `useLogin`)

## Добавление инпутов при регистрации

При отправке запроса в KeyCloak используется тег `<form action="...">`. На многошаговой регистрации возникает проблема: после перехода на следующий шаг предыдущая форма скрывается, и получить доступ к её инпутам невозможно.

**Решение:** на шаге ввода пароля добавлена скрытая форма с уже заполненными полями из предыдущих шагов. При нажатии на кнопку регистрации отправляется эта скрытая форма.

Это нужно для кейса с инвайт-хешем: хеш не отображается пользователю, но должен быть передан в KeyCloak.
